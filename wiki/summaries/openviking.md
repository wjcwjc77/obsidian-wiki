## Core points

- OpenViking presents a filesystem-style context database for agent memories, resources, and skills.
- It positions itself against fragmented context storage and opaque retrieval chains.
- Key ideas include tiered loading, recursive retrieval, observable retrieval traces, and automatic memory iteration.

## Why it matters

OpenViking strongly matches the design philosophy of this repo because it treats context as structured, inspectable, and evolvable.
